🏗️ Naxçıvan Residence ERP: Sistem Sxemi
1. Verilənlər Bazası Arxitekturası (SQL Server)
Sistem 4 ana blokdan və onları birləşdirən mərkəzi cədvəllərdən ibarət olacaq:
A. Aktivlər (Assets):
* Blocks: ID, Ad
* Apartments: ID, BlockID, No, Mertebe, Sahe, Status (Boş, Satılıb, Rezerv)
* Objects & Garages: ID, No, Sahe, Tip (Obyekt/Qaraj), Status (Boş, Satılıb, Kirayədə)
B. Müştəri və CRM:
* Customers: ID, Ad, Soyad, SHV_No, FIN, Mobil
* Users & Roles: Admin, Satis, Kommunal, Statistika, Idareetme (ASP.NET Identity)
C. Satış və Maliyyə:
* Sales: ID, CustomerID, AssetID, SatisTipi (Nağd/Kredit), UmumiMebleq, IlkinOdenis, Ay_Sayi
* Installments (Kredit qrafiki): ID, SaleID, OdenisTarixi, Mebleq, Status (Odenilib/Gozleyir)
* Rentals (Kirayə): ID, AssetID, CustomerID, Baslangic, Bitis, AyliqMebleq
D. Xidmətlər:
* Communal: ID, AssetID, Ay_Il, Borc ($Sahe \times Tarif$)
* Internet: ID, AssetID, PaketID, Status, BitisTarixi
2. Proqram Modulları və Funksionallıq
ModulƏsas FunksiyalarAdmin Paneliİşçilərin yaradılması, rolların verilməsi (RBAC), tariflərin (kvadrat metri, internet qiyməti) təyin edilməsi.Satış BölməsiMənzil/Obyekt seçimi, müştəri qeydiyyatı, faizsiz kredit hesablama və satışın təsdiqi.Kirayə ModuluBoş qalan obyekt və qarajların kirayə müqavilələrinin bağlanması və müddət izləmə.Kommunal & İnternetAylıq avtomatik borc yaratma, ödənişlərin qəbulu və qəbz (invoice) çıxarılması.Statistika & HesabatGəlir-çıxış qrafikləri, borclu siyahıları, satış faizi (neçə mənzil qaldı).
3. Satış və Kredit Hesablama Məntiqi (C# Logic)
Proqram daxilində kredit hesablama düsturu belə işləyəcək:
$$Ayliq\_Odenis = \frac{(Menzil\_Sahesi \times Bir\_kv\_Qiymeti) - Ilkin\_Odenis}{Kredit\_Muddeti (Ay)}$$
4. Developer üçün Hazır Prompt (ASP.NET & SQL)
Əgər bu proqramı yazdırmaq istəyirsənsə, aşağıdakı mətni proqramistə və ya AI-ya göndər:
Project Task: Create an ERP for 'Nakhchivan Residence' using ASP.NET Core MVC and SQL Server.
Core Logic Requirements:
1. Asset Management: Handle Apartments, Objects, and Garages with dynamic statuses (Available, Sold, Rented).
2. Sales Engine: Implement interest-free installment logic. Automatically generate an installment schedule (payment dates and amounts) upon sale.
3. Rental System: Manage lease agreements for unsold garages/objects (monthly/yearly).
4. Service Billing: > - Automatic monthly communal fees based on $m^2$.
   * Internet subscription management with automated billing cycles.
5. User Roles: RBAC (Admin, Sales, Utility, Stats, Management).
6. Technology Stack: EF Core (Code First), Identity Framework, Bootstrap for UI.
Deliverable: SQL Schema, C# Models, and main Controllers for Sales and Billing.
